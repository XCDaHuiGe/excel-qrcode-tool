# Excel 物资二维码批量生成工具产品需求

## 1. 背景与目标

用户需要从 Excel 表格中批量读取物资 ID，并为每条有效记录生成二维码标签。工具需要支持独立 PNG 批量导出，也需要支持 A4 PDF 打印排版。

本产品第一版定位为纯前端工具：Excel 解析、二维码生成、PNG 打包、PDF 排版和文件下载都在浏览器本地完成。后续如果出现上万行数据、历史记录、多用户管理或服务端存储需求，再扩展后端能力。

## 2. 用户与场景

目标用户是需要为物资、设备、库存或资产制作二维码标签的业务人员。

典型场景：

- 用户已有一份带表头的 Excel。
- Excel 中存在一列物资 ID。
- 用户希望批量生成二维码，并把二维码贴在物资上。
- 用户希望导出高清 PNG，用于单独使用或二次排版。
- 用户希望直接生成 A4 PDF，用普通打印机打印和裁切。

## 3. 范围

### 3.1 第一版包含

- 上传 Excel 文件。
- 解析并预览 Excel 前 20 行。
- 选择一个列作为物资 ID 列。
- 可选择一个列作为附加显示字段。
- 按 Excel 行顺序生成二维码记录。
- 空物资 ID 自动跳过。
- 重复物资 ID 不去重。
- 二维码内容第一版使用物资 ID 本身。
- 二维码标签显示二维码、物资 ID、可选附加字段。
- 批量导出 ZIP，内部为高清 PNG。
- 导出 A4 PDF。
- PDF 支持常用行列预设和自定义行列。
- PDF 自动计算标签尺寸并提供排版预览。
- PDF 支持可选裁切线，默认关闭。
- PDF 排版拥挤时提供分级风险提示。

### 3.2 第一版不包含

- 用户账号。
- 历史记录保存。
- 服务端上传和存储。
- 复杂资产管理。
- 多个附加字段显示。
- 手动设置单个标签宽高。
- 手动设置 PNG 像素尺寸。
- 直接连接数据库或业务系统。

## 4. 核心流程

1. 用户进入页面。
2. 用户上传 Excel 文件。
3. 系统解析文件并显示基础信息。
4. 系统展示前 20 行数据预览。
5. 用户选择物资 ID 列。
6. 用户可选一个附加显示字段列。
7. 系统生成二维码预览。
8. 用户选择导出方式：
   - 下载 ZIP。
   - 生成 PDF。
9. 如果生成 PDF，用户选择 A4 行列布局、裁切线等设置。
10. 系统展示 PDF 排版预览和拥挤风险。
11. 用户下载生成的文件。

## 5. Excel 上传与解析

### 5.1 文件要求

支持格式：

- `.xlsx`
- `.xls`
- `.csv`

Excel 必须包含表头。第一行固定视为表头，不提供“第一行是数据”的切换。

### 5.2 解析结果

文件解析成功后，系统显示：

- 文件名。
- 总行数。
- 总列数。
- 表头列表。
- 前 20 行数据预览。

### 5.3 数据预览

数据预览用于帮助用户确认选择正确的物资 ID 列和附加字段列。

预览规则：

- 最多展示前 20 行。
- 展示所有列或在列过多时横向滚动。
- 空值以明显但克制的状态展示，例如空白或浅色占位。

## 6. 列选择

### 6.1 物资 ID 列

用户必须选择一个物资 ID 列。该列用于：

- 二维码编码内容。
- 二维码标签第一行文字。
- PNG 文件名的一部分。

处理规则：

- 读取单元格展示值。
- 去除前后空格。
- 空值行跳过。
- 重复值不去重，按 Excel 行顺序保留。

### 6.2 附加显示字段

用户可以选择一个附加显示字段，也可以不选。

附加字段用于二维码标签第二行文字。

处理规则：

- 读取单元格展示值。
- 去除前后空格。
- 如果附加字段为空，则该标签不显示第二行。
- 附加字段不参与二维码编码。
- 附加字段不参与 PNG 文件名。

## 7. 二维码与标签

### 7.1 二维码内容

第一版二维码内容为物资 ID 本身。

产品需要在需求和结构上预留后续扩展：未来可支持二维码内容模板，例如：

```text
https://example.com/material/{物资ID}
```

第一版界面可以不暴露模板配置，但数据模型和生成逻辑不应把二维码内容与显示文字写死为不可扩展结构。

### 7.2 标签视觉结构

每个标签包含：

- 上方二维码。
- 下方第一行：物资 ID。
- 下方第二行：附加字段，只有存在值时显示。

基本要求：

- 白底黑码。
- 二维码居中。
- 文字居中。
- 标签四周有基础留白。
- 文字过长时应自动缩小或截断提示，避免溢出到其他标签。

## 8. ZIP 与 PNG 导出

### 8.1 导出行为

用户点击下载 ZIP 后，系统为每条有效记录生成一张高清 PNG，并打包成 ZIP。

PNG 尺寸第一版固定，不提供用户配置。

### 8.2 文件命名

文件名格式：

```text
序号_物资ID.png
```

示例：

```text
001_WZ001.png
002_WZ001.png
003_WZ002.png
```

命名规则：

- 序号按有效记录顺序生成。
- 序号自动补零，位数根据总数量决定，最少 3 位。
- 物资 ID 中不适合文件名的字符替换为 `_`。
- 如果替换后文件名仍冲突，系统追加额外序号保证唯一。

## 9. PDF 导出

### 9.1 页面规格

PDF 使用 A4 纸张。

第一版默认纵向。横向可以作为后续扩展，不作为第一版必需能力。

### 9.2 行列布局

用户可以选择常用预设，也可以自定义行列。

第一版预设：

- 2 列 x 4 行，8 个每页。
- 3 列 x 3 行，9 个每页。
- 3 列 x 4 行，12 个每页。
- 4 列 x 5 行，20 个每页。

自定义规则：

- 用户输入列数和行数。
- 系统根据 A4 可用区域自动计算每个标签单元大小。
- 标签在每个单元内自动适配。
- 系统自动分页。

### 9.3 排版预览

PDF 生成前必须提供排版预览。

预览内容：

- A4 页面缩略预览。
- 当前行列布局。
- 单页可容纳数量。
- 总页数。
- 前若干个真实二维码标签或代表性标签。
- 是否开启裁切线。
- 拥挤风险状态。

### 9.4 裁切线

裁切线为可选项，默认关闭。

开启后：

- 在标签单元边界显示浅色分隔线。
- 分隔线不应压住二维码主体。
- 分隔线颜色应适合打印，避免过深。

## 10. 拥挤检测

用户设置的行列过密时，系统需要给出分级提示。

### 10.1 检测维度

检测重点包括：

- 二维码实际显示尺寸是否过小。
- 物资 ID 是否能完整显示。
- 附加字段是否能完整显示。
- 两行文字是否挤压二维码区域。
- 标签上下左右留白是否过小。

### 10.2 风险等级

无风险：

- 二维码边长不小于 25 mm。
- 文字可正常显示。
- 留白合理。
- 允许生成。

轻微拥挤：

- 二维码边长大于等于 20 mm 且小于 25 mm，或文字接近可显示宽度上限。
- 页面提示用户打印前确认扫码和文字可读性。
- 允许生成。

严重拥挤：

- 二维码边长小于 20 mm，或物资 ID 文字无法在一行内正常显示。
- 阻止生成 PDF。
- 提示用户减少行数或列数。

附加字段过长时优先触发轻微拥挤提示；如果第二行在缩小字体后仍无法显示，则触发严重拥挤。

## 11. 状态与错误处理

### 11.1 上传错误

需要处理：

- 文件格式不支持。
- 文件为空。
- 文件解析失败。
- 没有表头。
- 没有数据行。

### 11.2 数据错误

需要处理：

- 未选择物资 ID 列。
- 物资 ID 列全部为空。
- 部分物资 ID 为空。
- 附加字段为空。
- 物资 ID 过长。
- 附加字段过长。

### 11.3 导出错误

需要处理：

- 没有有效二维码记录。
- ZIP 生成失败。
- PDF 生成失败。
- PDF 布局严重拥挤。

## 12. 页面信息架构

页面分为 4 个主要区域。

### 12.1 上传区

包含：

- 文件上传入口。
- 支持格式说明。
- 当前文件名。
- 解析状态。

### 12.2 数据配置区

包含：

- 物资 ID 列选择。
- 附加字段列选择。
- 有效记录数量。
- 跳过空 ID 行数。
- Excel 前 20 行预览。

### 12.3 二维码预览区

包含：

- 前若干个二维码标签预览。
- 物资 ID 和附加字段实际显示效果。
- 数据量较多时限制预览数量，避免页面卡顿。

### 12.4 导出区

包含：

- 下载 ZIP 按钮。
- PDF 行列预设。
- 自定义行数和列数。
- 裁切线开关。
- PDF 排版预览。
- 下载 PDF 按钮。
- 拥挤风险提示。

## 13. 默认设置

- 二维码内容：物资 ID。
- 重复物资 ID：不去重。
- 空物资 ID：跳过。
- 附加字段：默认不选。
- 附加字段为空：不显示第二行。
- ZIP PNG 尺寸：固定高清。
- PNG 文件名：序号_物资ID.png。
- PDF 纸张：A4 纵向。
- PDF 默认布局：3 列 x 4 行。
- 裁切线：默认关闭。

## 14. 验收标准

1. 上传带表头 Excel 后，系统能正确解析表头和数据。
2. 系统能展示前 20 行数据预览。
3. 用户必须选择物资 ID 列后才能生成二维码。
4. 用户可以选择一个附加字段列，也可以不选。
5. 空物资 ID 行会被跳过，并显示跳过数量。
6. 重复物资 ID 会按 Excel 行顺序保留并生成多张标签。
7. 二维码编码内容为物资 ID。
8. 标签下方第一行显示物资 ID。
9. 标签下方第二行显示附加字段，附加字段为空时不显示第二行。
10. ZIP 导出包含所有有效记录的 PNG。
11. PNG 文件名使用序号加物资 ID，并能处理重复和非法字符。
12. PDF 导出支持常用行列预设。
13. PDF 导出支持自定义行数和列数。
14. PDF 预览能展示 A4 排版、单页数量、总页数和拥挤风险。
15. 裁切线可选，默认关闭。
16. 轻微拥挤时允许生成并提示风险。
17. 严重拥挤时阻止生成 PDF 并提示调整行列。
18. 全流程可在浏览器纯前端完成。
