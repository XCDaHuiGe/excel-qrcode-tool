# Excel 物资二维码批量生成工具 Stitch UI 设计提示词

## 1. 可直接粘贴给 Stitch 的完整提示词

请设计一个高保真的桌面端 Web 工具界面，产品名称是「Excel 物资二维码批量生成工具」。

这个产品不是营销网站，也不是展示型首页，而是一个真实可用的企业工具型前端页面。用户进入页面后，第一屏就应该能完成主要操作：上传 Excel、选择字段、预览二维码标签、配置 PDF 排版、导出 ZIP 或 PDF。

### 产品用途

用户上传一个带表头的 Excel 文件，选择其中一列作为「物资 ID 列」。系统根据每一行物资 ID 批量生成二维码标签。

每个二维码标签的结构是：

- 上方：二维码。
- 下方第一行：物资 ID。
- 下方第二行：可选附加字段，例如物资名称、规格、位置。

系统支持两种导出方式：

- 导出 ZIP：每个物资 ID 生成一张高清 PNG，文件名格式为「序号_物资ID.png」。
- 导出 PDF：生成 A4 打印排版文件，用户可以选择每页几行几列。

### 视觉方向

整体风格请采用「科技感企业操作台」。

关键词：

- 高科技企业 SaaS 控制台。
- 物流资产管理后台。
- 数据处理工作台。
- 打印排版工具。
- 深色顶部命令栏。
- 浅色主工作区。
- 电光蓝 / 青色强调色。
- 细网格线。
- 克制、专业、清晰、可长时间使用。

不要做成营销页，不要出现大 Hero 区，不要有大段宣传文案。

不要使用：

- 大面积渐变背景。
- 装饰性光球。
- 夸张插画。
- 卡片堆叠式营销布局。
- 过大的标题。
- 过度圆角。
- 纯展示用的假按钮。

需要让界面看起来是一个可以真实开发和使用的产品。

### 画布要求

- 输出 16:9 桌面端 UI 设计稿。
- 推荐尺寸：1440 x 900 或 1600 x 1000。
- 中文界面。
- 高保真。
- 所有主要控件都需要可真实实现。
- 第一屏内展示完整工作流。

## 2. 页面整体布局

页面采用「顶部命令栏 + 步骤条 + 三栏工作台」结构。

### 2.1 顶部命令栏

位置：页面最上方。

高度：约 56 px。

背景：深石墨色或近黑色，例如 `#101827`、`#111827`、`#0F172A`。

左侧内容：

- 产品图标：可以是简洁的二维码 / 文件 / 扫码类线性图标。
- 产品名称：`Excel 物资二维码批量生成工具`。
- 产品名称字体需要清晰，不要过大，约 16-18 px。

右侧内容：

- 状态标签：`本地处理`。
- 状态标签样式：小号胶囊标签，青色描边或青色文字。
- 文件状态：未上传时显示 `未上传文件`，上传后显示文件名缩略，例如 `materials_2026.xlsx`。
- 可选一个轻量按钮：`清空重置`。

顶部命令栏按钮规范：

- `清空重置`：次级按钮，深色背景内使用描边或幽灵按钮样式。
- 鼠标悬停态需要轻微高亮。
- 不要使用大面积填充色。

### 2.2 顶部步骤条

位置：命令栏下方。

高度：约 48-56 px。

背景：白色或极浅灰。

步骤内容：

1. `上传 Excel`
2. `选择字段`
3. `预览二维码`
4. `导出文件`

步骤条规范：

- 使用横向 stepper。
- 当前步骤使用电光蓝强调色。
- 已完成步骤可以使用蓝色圆点或勾选图标。
- 未完成步骤使用灰色。
- 步骤之间使用细线连接。
- 不要做得像流程海报，要像后台工具的进度导航。

### 2.3 主体三栏工作台

主体区域从左到右分为三栏：

1. 左侧：Excel 上传与数据预览。
2. 中间：二维码标签预览。
3. 右侧：导出与 PDF 设置。

建议宽度比例：

- 左侧：32%。
- 中间：38%。
- 右侧：30%。

主体区域背景：

- 页面底色为浅灰，例如 `#F4F7FB` 或 `#F6F8FC`。
- 面板为白色或接近白色。
- 面板边框为浅灰蓝，例如 `#DDE5F0`。
- 面板圆角 8 px 或更小。
- 不要使用过大的 20 px 圆角。

## 3. 左侧面板：Excel 上传与字段配置

### 3.1 面板标题区

标题：`Excel 数据源`

副信息：`上传带表头的 Excel 文件，预览前 20 行`

右上角可显示解析状态：

- 未上传：`等待上传`
- 解析中：`解析中`
- 已完成：`已解析`
- 出错：`解析失败`

状态样式：

- 未上传：灰色。
- 解析中：蓝色。
- 已完成：绿色或青色。
- 出错：红色。

### 3.2 上传文件区域

控件类型：拖拽上传区域 + 按钮。

默认状态内容：

- 图标：文件上传图标。
- 主文案：`拖拽 Excel 到这里`
- 辅助文案：`支持 .xlsx / .xls / .csv`
- 主按钮：`选择文件`

上传区域样式：

- 白色或极浅蓝背景。
- 虚线边框。
- 悬停时边框变为电光蓝。
- 上传后区域收起为文件信息卡。

上传后文件信息卡内容：

- 文件名：例如 `物资清单.xlsx`
- 文件大小：例如 `128 KB`
- 数据统计：`共 236 行 · 8 列`
- 操作按钮：`更换文件`

按钮规范：

- `选择文件`：主按钮，电光蓝填充。
- `更换文件`：次级按钮，白底蓝色描边。
- 按钮高度约 36-40 px。
- 按钮圆角 6-8 px。
- 按钮文字 14 px。

### 3.3 字段选择区

标题：`字段映射`

字段 1：

- Label：`物资 ID 列`
- 控件：下拉选择框。
- Placeholder：`请选择物资 ID 列`
- 必填标记：红色星号或必填提示。
- 示例选项：`物资编码`、`资产编号`、`设备ID`、`Material ID`。

字段 2：

- Label：`附加显示字段`
- 控件：下拉选择框。
- Placeholder：`可选：名称 / 规格 / 位置`
- 非必填。
- 下拉第一项：`不显示附加字段`。

字段选择区提示：

- 物资 ID 会写入二维码内容。
- 附加字段只显示在标签第二行，不参与二维码编码。

字段选择区按钮：

- `应用字段`：主按钮或小型强调按钮。
- 如果物资 ID 列未选择，按钮禁用。

禁用态规范：

- 背景灰色。
- 文字灰色。
- 不可点击。
- 鼠标悬停不变化。

### 3.4 数据统计条

在字段选择后显示三个小统计项：

- `有效记录`：例如 `228`
- `跳过空 ID`：例如 `8`
- `重复 ID`：例如 `12`

注意：重复 ID 不去重，只统计提示。

统计卡样式：

- 三个小块横向排列。
- 白底或浅蓝底。
- 数字较醒目。
- 标签文字较小。

### 3.5 Excel 表格预览

标题：`数据预览`

说明：`仅展示前 20 行`

表格列示例：

- `物资编码`
- `物资名称`
- `规格型号`
- `存放位置`
- `数量`

表格行示例：

- `WZ-2026-001`、`工业传感器`、`S-220`、`A区-03架`、`12`
- `WZ-2026-002`、`控制模块`、`CM-18`、`B区-01架`、`4`

表格规范：

- 表头固定在顶部。
- 表格高度有限，内部滚动。
- 行高约 36-40 px。
- 字号 12-13 px。
- 鼠标悬停行高亮。
- 被选为物资 ID 的列可以用浅蓝背景标记。
- 被选为附加字段的列可以用浅青背景标记。

## 4. 中间面板：二维码标签预览

### 4.1 面板标题区

标题：`二维码标签预览`

副信息：`按 Excel 行顺序生成，重复 ID 保留`

右上角显示：

- `预览 12 / 228`

### 4.2 工具条

工具条包含：

- 视图切换按钮：`网格`、`单页`
- 缩放控制：`-`、`100%`、`+`
- 可选按钮：`刷新预览`

按钮规范：

- `网格 / 单页` 使用分段控制。
- 当前选中项使用蓝色背景或蓝色边框。
- 缩放按钮使用图标按钮，不要写成长按钮。

### 4.3 二维码标签卡片

卡片内容：

- 上方二维码图形。
- 下方第一行：物资 ID，例如 `WZ-2026-001`。
- 下方第二行：附加字段，例如 `工业传感器`。

卡片尺寸：

- 预览状态下约 132 x 170 px。
- 保持固定比例。
- 卡片之间间距 12-16 px。

卡片视觉：

- 白底。
- 浅灰边框。
- 轻微阴影。
- 二维码区域居中。
- 下方文字居中。
- 第二行文字颜色稍浅。

二维码视觉：

- 可以使用真实二维码占位图形。
- 不要用普通方块代替。
- 二维码应显得真实可扫。

标签状态：

- 正常：白底。
- 附加字段为空：只显示一行物资 ID。
- 文字过长：显示压缩或省略效果，但需要有风险提示。

### 4.4 空状态

如果未上传文件：

- 显示空状态图标。
- 文案：`上传 Excel 后将在这里预览二维码标签`
- 次级文案：`支持批量生成 PNG 和 A4 PDF`

空状态不要使用夸张插画，可以使用线性图标和轻科技感背景。

## 5. 右侧面板：导出与 PDF 设置

### 5.1 面板标题区

标题：`导出设置`

副信息：`生成 ZIP 图片包或 A4 PDF`

### 5.2 ZIP 导出区

模块标题：`PNG 图片包`

说明文案：

- `每条有效记录生成一张高清 PNG`
- `文件名：序号_物资ID.png`

按钮：

- 主按钮：`下载 ZIP`

按钮状态：

- 无有效记录：禁用。
- 生成中：显示 loading，文案为 `生成中...`
- 可下载：电光蓝填充。

按钮旁可显示：

- `预计 228 张 PNG`

### 5.3 PDF 导出区

模块标题：`A4 PDF 排版`

纸张信息：

- `纸张：A4 纵向`
- `自动分页`

常用预设：

- `2×4`
- `3×3`
- `3×4`
- `4×5`

预设控件规范：

- 使用分段按钮或小型选择按钮。
- 默认选中 `3×4`。
- 选中态使用蓝色描边或蓝色底。

自定义行列：

- Label：`自定义行列`
- 输入框 1：`列数`
- 输入框 2：`行数`
- 输入框应为数字输入。
- 输入范围建议：列数 1-6，行数 1-10。

辅助显示：

- `每页数量：12`
- `总页数：19`

### 5.4 裁切线开关

控件类型：toggle switch。

Label：`显示裁切线`

默认：关闭。

说明：`适合普通 A4 纸裁剪，贴纸纸可关闭`

开关规范：

- 关闭态：灰色。
- 开启态：电光蓝。

### 5.5 拥挤风险提示

模块标题：`排版风险`

风险状态有三种：

1. 正常
   - 文案：`当前布局适合打印`
   - 颜色：绿色或青色。

2. 轻微拥挤
   - 文案：`二维码或文字接近推荐下限，请打印前确认可扫码`
   - 颜色：橙色。
   - 允许导出 PDF。

3. 严重拥挤
   - 文案：`当前行列过密，二维码或文字可能无法正常使用`
   - 颜色：红色。
   - PDF 下载按钮禁用。
   - 提示操作：`请减少行数或列数`

风险提示样式：

- 使用小型状态卡。
- 左侧可有状态图标。
- 不要弹窗打断用户。

### 5.6 PDF 下载按钮

主按钮：`下载 PDF`

按钮状态：

- 无有效记录：禁用。
- 严重拥挤：禁用。
- 生成中：loading，文案 `生成 PDF 中...`
- 正常可下载：电光蓝填充。

旁边可有次级按钮：

- `预览 PDF`

如果空间不足，可以只保留 `下载 PDF`。

## 6. 交互状态要求

请在设计稿中体现或预留以下状态。

### 6.1 初始状态

- 未上传文件。
- 左侧显示上传区域。
- 中间显示二维码预览空状态。
- 右侧导出按钮禁用。
- 步骤条停留在「上传 Excel」。

### 6.2 上传后状态

- 左侧显示文件信息和表格预览。
- 字段下拉可选。
- 步骤条进入「选择字段」。
- 中间仍可提示选择字段后预览。

### 6.3 已选择字段状态

- 显示有效记录、跳过空 ID、重复 ID。
- 中间显示二维码标签预览。
- 右侧导出按钮可用。
- 步骤条进入「预览二维码」。

### 6.4 PDF 设置状态

- 右侧可以选择 2×4、3×3、3×4、4×5。
- 自定义行列改变后，单页数量和总页数实时更新。
- 拥挤风险实时变化。

### 6.5 导出中状态

- `下载 ZIP` 或 `下载 PDF` 按钮显示 loading。
- 页面不要整体遮罩。
- 可在按钮下方显示进度，例如 `正在生成 36 / 228`。

## 12. 负面提示词

- 营销首页。
- 大 Hero 区。
- 人物插画。
- 夸张 3D 插画。
- 大面积渐变背景。
- 装饰性光球。
- 无意义的统计图表。

- 过度圆角卡片。

- 看起来不能真实开发的控件。
- 英文为主的界面。
